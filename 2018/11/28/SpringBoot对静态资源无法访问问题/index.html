SpringBoot对静态资源无法访问问题

2018-11-28 评论

问题

今天在学习SpringBoot-拦截器项目学习中遇到了拦截器拦截静态资源(.css,.js)问题。导致网页中的没有样式

在SpringBoot官方文档中指出将静态资源放在指定的路径下会自动做好静态资源的映射。类路径下（resources包下的
SpringBoot对静态资源的默认扫描路径:

  • classpath:/static
  • classpath:/public
  • classpath:/resources
  • classpath:/META-INF/resources


我项目静态资源位置如下图



个人表示已经放在SpringBoot指定的路径下了，还是进行了拦截！！！

解决问题

后来阅读大量的博客和解决办法，是因为SpringBooot2.0以后的版本对静态资源进行了拦截！真的是这样末？？？？个人觉得应该不是这样吧！！不能在版本更新的过程中，将上个版本的优点摸掉吧！应该取长补短吧！！！
目前我的解决办法就是对改路径下的所有静态资源进行放行操作！
‘’’

//注册拦截器
@Override
public void addInterceptors(InterceptorRegistry registry) {
    //super.addInterceptors(registry);
    //"/**"表示拦截所有
    registry.addInterceptor(new LoginHandlerInterceptor()).addPathPatterns("/**").excludePathPatterns("/login.html","/","/user/login","/asserts/**","/webjars/**");

对类路径下的asserts,webjars包里的所有放行！
这样就对静态资源放行了！！！刷新页面可看到
